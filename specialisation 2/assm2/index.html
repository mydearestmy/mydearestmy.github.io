Orbital Sound – UI Progress

Open Orbital Sound Prototype

Conceptual Response

For the prompt about “spatial control + randomness Math.random(),” I’m developing Orbital Sound, an experimental audio interface. A small moon orbits a central planet:

  • Distance controls playback speed
  • Angle sets stereo pan

The goal is to make sound control feel physical and playful, rather than using standard sliders.

Design References

  • NASA Eyes on Solar System – interactive planetary visualization and orbit metaphor.

Visual Plan & System Map

  • Planet (centre): anchor point and audio emitter
  • Moon (draggable): user input, sends position data
  • Math layer: calculates radius + angle from drag coordinates
  • Audio engine: maps radius → playback rate, angle → pan
  • Feedback layer: colour, trail or shadow to show changes
  • Onboarding: subtle pulse or “drag here” arrow on load

Technical Approach

  • Built in HTML, CSS, and vanilla JavaScript
  • Audio handled via the Web Audio API
  • pointerdown / pointermove events track the moon’s location
  • requestAnimationFrame keeps orbit motion smooth
  • Logic separated into modules: visuals, audio, interaction
  • Planned upgrade: stronger easing + hint animation for onboarding

Reflection on Week 8 Peer User Testing

Prompts Shown to Peers

  • “Was it obvious you could drag the small circle?”
  • “What would make the control more engaging?”
  • “Final thoughts or advice?”

What People Said

  • Many eventually worked out the moon was draggable, but some weren’t sure at first.
  • Hover cursor and colour helped, yet a clear drag cue was requested.
  • Several felt the centre planet’s role was unclear.
  • Audio feedback was too subtle; they wanted bigger sonic shifts.
  • Suggestions: stronger colours while dragging, shadow or trail to show pan/speed, extra circle for variety.

My Response & Next Steps

  • Add onboarding hint (pulse or arrow) on first load
  • Give the planet a subtle glow or label to explain its purpose
  • Emphasise extremes: richer colour/audio when orbiting far/near
  • Prototype shadow on the planet for pan and a trail to signal speed
  • Tweak audio layers for a more noticeable, rewarding change
